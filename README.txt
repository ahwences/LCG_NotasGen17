Notas colaborativas de las clases de Estadística y R hechas por los alumnos. El sistema de control de versiones escogido es git, con acceso desde github, para que además los alumnos se entrenen en la escritura de código/documentos de manera colaborativa utilizando herramientas formales. La regla más importante es que cada vez que hagan un commit escriban una descripción acertada de los cambios que hicieron, y además que traten de modificar los archivos de poco en poco, haciendo los commits pertinentes para que el historial de cambios sea lo más claro (y al mismo tiempo conciso) posible. Los profesores sólo intervendrán en el documento para verificar el contenido de las notas y de ser necesario señalar y/o realizar posibles correcciones. 

Recuerda que los pasos para empezar a colaborar son:
1) De preferencia actualizar tu versión local del repositorio utilizando 'git pull origin main'.
2) Realizar cambios locales en los archivos.
3) Anunciar a git los cambios que queremos guardar utilizando el commando 'git add', por ejemplo para anunciar TODOS los cambios hechos (incluyendo la adición de todos los nuevos archivos encontrados) puedes usar 'git add .'
4) Hacer un commit de los cambios anunciados en el paso anterior dentro de tu repositorio local con 'git commit -m *mensaje*'.
5) Publicar tus cambios en el repositorio de github con 'git push' para que todos los puedan ver.
